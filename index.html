In this page, you can try ShadowBlock's blocking capabilities. Please note that the extension does not have any code that is specific to this page. It runs the same code that is used in ygosu.com and ppss.kr.

There are two images below, the red one is an ordinary image, and the blue one is an image that uses "shadow DOM".
Also, below the second, blue image, there is a message indicating this site's attempt to detect whether the image is being hidden or not. If you have done nothing for this page yet, it should be saying "Blue image is not being blocked".

Most adblockers have "Element picker" tool that allows you to pick an element in a page to block it.

  • Adblock Plus: https://help.eyeo.com/adblockplus/block-item-or-element
  • uBlock Origin: https://github.com/gorhill/uBlock/wiki/Element-picker
  • AdGuard: https://kb.adguard.com/en/browser-extensions/overview.html#assistant

1. Try to block the ordinary(red) image with those tools. There should be no problem in doing so.

2. Now try to block the shadow(blue) image with those tools. Some tools may be unable to do so, while some can. If it can't, add this adblocking rule by hand to your adblocker's filter: shadow-b1ock.github.io##img Then the message below the blue image will be changed to "Blue image is being blocked".

3. Now time to see the difference. Remove all the previously-installed extensions and install ShadowBlock, and refresh the page.

You should be able to observe that the blue image is now being blocked, and the message below it still indicates that "Blue image is not being blocked".

This blue image is a simplified version of what certain ads recently started to appear in the supported websites uses.
